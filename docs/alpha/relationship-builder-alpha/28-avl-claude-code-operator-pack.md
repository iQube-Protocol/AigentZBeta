# AVL Relationship Builder — Claude Code Operator Pack

## Build mandate

Claude Code is the exclusive implementation lead for AVL Relationship Builder.

### Core build decision
- replace the current user-facing Marketa Cartridge concept with AVL Relationship Builder
- keep Marketa as the lead relationship agent
- make KNYT Relationship Composer the first live program workspace
- treat the live Kickstarter as Campaign Wave 1 of the ongoing KNYT Program
- build for continuity beyond Kickstarter close

## Do not
- revive the old Lovable portal as the production app
- build a separate parallel product outside the current Next.js stack
- frame Make.com as the product
- treat KNYT as a one-off campaign that ends at Kickstarter close

## Do
- preserve current AigentZBeta stack gravity
- use the old portal as workflow and IA reference only
- integrate Studio, Experience Matrix, Registry, Ingestion Factory, QubeTalk, CRM, and Make-as-rail-1 into one relationship orchestration system
- support both partner and customer relationship management inside AVL
- model Program -> Campaign Wave -> Always-On Action

## Immediate implementation priorities
1. audit current Marketa implementation and map it to AVL
2. reframe product IA around internal and partner modes
3. add the internal customer surface
4. add KNYT as first-class Program
5. add KNYT Relationship Composer
6. support seeded KS Backers cohort activation via email-first execution
7. implement Stage 1.5 hygiene and canonization gate
8. prepare Stage 2 cohort expansion and enrichment
