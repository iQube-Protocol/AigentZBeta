# The Constitutional Internet — Structured Report

**EXP-001 artifact 2 of 5 · Report format · Grounded exclusively in KnowledgeQube 001 (markers `[C-NNN]`)**

## 1. Executive summary

The Constitutional Internet is a network architecture in which legitimacy is structural: persons precede identifiers [C-011], sovereignty is human-only [C-016], authority is delegable but bounded [C-015, C-020], power is calibrated by validated Standing [C-013], and truth, standing, and reach are held as orthogonal dimensions [C-060–062]. Its record is append-only [C-023] and privacy-preserving [C-024]; its semantics are human-ratified [C-021, C-022].

## 2. Constitutional ordering

| # | Ordering | Invariant |
|---|---|---|
| 1 | Person → identity | Personhood precedes identity. [C-011] |
| 2 | Action → standing | Standing follows action. [C-012] |
| 3 | Standing → authority | Authority follows standing. [C-013] |
| 4 | Validation → truth-claim | Truth is established through validation within a domain of applicability, not by popularity. [C-060] |

The orderings compose into one chain: person → action → standing → authority, with truth-claims admitted only through validation.

## 3. Agency model

| Rule | Invariant |
|---|---|
| Delegation permitted; sovereignty non-transferable | [C-015] |
| Sovereignty is exclusively human | [C-016] |
| Agents cannot mint authority | [C-017] |
| Accountability survives delegation | [C-014] |
| No permanent/unlimited delegation | [C-020] |

**Implication:** agents are first-class actors with real (borrowed) power and zero constitutional personhood. Every agent act has an accountable human principal [C-014].

## 4. The Standing system

- Definition: confidence in the veracity of declarations — explicitly not reputation, popularity, or status [C-018].
- Accrual: only through action and subsequent validation [C-012].
- Citizen obligation: veracity of declarations, not consequence prediction [C-019].
- Function: the calibration input for authority [C-013].
- Scope: extends to knowledge itself — invariants accrue Standing [C-059].

## 5. The three orthogonal dimensions (risk register framing)

| Dimension | Measures | Failure if conflated |
|---|---|---|
| Truth | Validated accuracy within a domain [C-060] | Popularity mistaken for fact |
| Standing | Accumulated constitutional confidence [C-061] | Confidence mistaken for truth itself |
| Reach | Adoption/usage [C-062] | Virality mistaken for validity |

Conflation of any pair is a constitutional defect [C-062].

## 6. Record & privacy guarantees

- **Append-only memory**: supersession, never deletion; superseded records remain auditable [C-023].
- **Subject protection**: re-identifying identifiers never leave the server [C-024].

## 7. Human/machine boundary

- Semantics (meaning, canon) are defined by humans [C-021].
- Canonical admission requires human ratification [C-022].
- Machines optimize implementation within human-defined meaning [C-021].

## 8. Conclusions

1. Legitimacy is an ordering problem; the Constitutional Internet fixes the order (person → action → standing → authority) before any application logic. [C-011, C-012, C-013]
2. Agency scales safely only under bounded, accountable delegation. [C-014, C-015, C-017, C-020]
3. An epistemically honest network requires the Truth/Standing/Reach separation as an architectural property, not a policy. [C-060, C-061, C-062]
4. Institutional trust requires memory that cannot be quietly rewritten. [C-023, C-024]
