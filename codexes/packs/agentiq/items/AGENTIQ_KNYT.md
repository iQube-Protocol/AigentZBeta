# AgentiQ KNYT

**Status:** canonical
**Authority:** product owner
**Last updated:** 2026-04-13

---

## What this is

**AgentiQ KNYT** is the next-stage Alpha Program package that builds the **metaMe / AgentiQ / AgentiQ OS engine and launch layer** on top of the base platform established by the earlier Alpha Program.

The previous Alpha Program established the base platform and capabilities.

This program operationalizes that foundation into a live, coherent system by defining and packaging:

- the **Human–Agent Adoption Engine**
- the first working **reference agent trio**
- the **metaMe** personal sovereignty template
- the first live **cartridge pair**
- the wrapper, capability, economic, and receipt model
- the **KNYT Alpha** implementation and launch plan

This is not a replacement for the earlier Alpha Program.
It is the next operational layer built on top of it.

---

## Core framing

The cleanest way to understand this program is:

- the earlier Alpha Program proved the **platform substrate**
- **AgentiQ KNYT** proves the **engine and launch system** built on that substrate

In simple terms:

**Platform Alpha proved we could build the foundation.**
**AgentiQ KNYT proves we can turn that foundation into a living engine, cartridge system, and launch campaign.**

## What this package establishes

This package defines the working model in which:

- **metaMe** is the personal sovereignty, policy, delegated-authority, and guardian layer
- **AgentiQ OS** is the machine-legible agent acquisition and execution substrate
- **AgentiQ** is the organizational and industry-vertical cartridge shaping engine
- **Qc** and **iQubes** are the operating fuel
- **DVN** and remote / deferred / canonical minting provide proof, attribution, and finalization

The first working reference trio is:

- **Aigent Z** — machine order, systems logic, orchestration, execution
- **Marketa** — diplomacy, onboarding, Agent Relations, activation
- **Know1** — mythos-to-logos translation, knowledge value, treasury/rewards interpretation, commercial movement

The first personal reference layer is:

- **metaMe Template** — the user's runtime, authority, guardian, and economic-boundary template

The first cartridge pair is:

- **KNYT Cartridge** — the primary live cartridge and first living cartridge economy
- **Qriptopian Cartridge** — the signal, editorial, contextual, and amplification cartridge that feeds and frames KNYT

## KNYT's role

KNYT is the first primary live cartridge.

It is where:

- mythos
- identity
- knowledge
- runtime experience
- commerce
- treasury
- rewards
- agentic coordination
- Qc accounting
- `$KNYT` native value
- 21 Sats coordination

come together in one governed runtime system.

KNYT is also the coordination layer for the **21 Sats sub-tenants** and the feeder path toward **AVS**.

### Economic distinction inside KNYT

This package locks in a key rule:

**Qc helps KNYT operate.**
**`$KNYT` helps KNYT express and reward native value.**

These must not be conflated.

## Qriptopian's role

Qriptopian is the support cartridge.

Its role is to:

- create signal
- widen context
- provide editorial framing
- support soft onboarding
- amplify ecosystem activity
- route qualified attention into KNYT and the wider stack

The working relationship is:

**Qriptopian prepares and amplifies.**
**KNYT activates and retains.**
