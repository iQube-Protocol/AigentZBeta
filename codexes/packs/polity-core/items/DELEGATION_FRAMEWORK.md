# Delegation Framework

**Version 1.0.0 · Ratified 2026-06-17 · Binds Constitution 1.0.0**

Delegation is how a sovereign citizen lends bounded authority to an agent
without ever lending sovereignty. Every delegation is a sealed envelope, bounded
on every dimension, and immutable once created.

## Bounded on every dimension

| Dimension | Meaning |
|---|---|
| **Scope** | The explicit set of allowed actions. |
| **Duration** | A finite TTL. No permanent grants. |
| **Spending limits** | A capped economic envelope. Assets remain the sponsor's. |
| **Information access limits** | A disclosure class bounding what the agent may read. |
| **Operational domains** | The surfaces / cartridges the agent may act within. |

## Prohibitions

**Permanent or unlimited delegation is prohibited.** A delegation that is
missing any bound, or that grants unbounded scope, duration, spend, access, or
domain, is invalid.

## Immutability

The delegation envelope is **immutable after creation**. No conversation and no
agent action may expand it. Expansion requires a new, explicit grant from the
sponsor.

## Relationship to bounded delegation today

The existing bounded-delegation surface (trust bands, TTL, action limits,
disclosure class, allowed surfaces, spend autonomy) is the citizen-facing
expression of this framework. Option A autonomous agents operate under the same
envelope model, with the additional Agent Charter constraints.
