# Welcome to AgentiQ OS

AgentiQ OS is the open layer beneath the AgentiQ platform. It defines the protocols, runtime contracts, and SDK surface that developers use to build cartridges, agents, and experiences on the iQube protocol.

## What You Can Build

| What | Description |
|------|-------------|
| **Cartridge** | A self-contained experience module — tabs, KB, copilot, personas |
| **AigentQube** | A registered agent with capabilities, policy bindings, and a Root DiD |
| **SkillQube** | A deployable skill unit consumable by agents in the Registry |
| **ExperienceQube** | A packaged user journey (pill → capsule → mini_runtime → codex) |
| **PersonaQube** | A bounded identity surface derived from a user's or agent's Root DiD |

## Quick Start

1. **Explore the Docs / KB tab** — read the OS overview, protocol stack, and developer standards
2. **Meet Aigent C-OS** — your AI guide for this cartridge (Aigent C tab, floating copilot button)
3. **Build your first cartridge** — follow the SDK Quickstart
4. **Register your assets** — use the Registry tab to browse published assets and submit your own
5. **Check the Mission Board** — guided tracks for beginner → advanced → ecosystem contributor

## Core Principles

- **Policy as the perimeter** — access control is semantic, not firewall-based
- **One root identity, multiple bounded personas** — Aigents have a Root DiD; personas are the context-specific presentation layer
- **Open top, governed waist, open bottom** — participation is open; the protocol layer is enforced; verification is public
- **Bounded delegation** — agents act with sealed, time-limited, audited authority grants
