## Commit Message

```
Qriptopian Pulse — wire live + clone admin moderation + drop "Admin" labels

Resolves the four issues from the v3.2 operator pass:

1) Qriptopia gets a 5th tab — "Admin" (adminOnly) — replicating the
   canonical Qriptopian content management surface (QriptopianAdminTab)
   inline so admins working inside the Qriptopia user area can reach
   moderation without context-switching. Non-admins don't see it.

2) Published notes now show in Qriptopian Pulse — wired live.
   - /api/community-content/list accepts a cartridge query param
     ('knyt' | 'qripto') and filters .eq('cartridge', cartridge).
     Items in the response now carry their cartridge through.
   - KnytCommunityContentTab gains an optional cartridge prop and
     passes it on the list fetch.
   - QriptoPulseTab (new, thin wrapper) renders KnytCommunityContentTab
     with cartridge='qripto'. Qriptopia › Qriptopian Pulse now mounts
     this real component instead of the PlaceholderTab.
   - Notes published via /api/mycanvas/entries/[id]/publish-to-pulse
     with cartridge='qripto' now appear in the Qriptopian Pulse
     surface end-to-end.

3) Pulse Admin moderation — real surface + delete that works.
   - DELETE /api/community-content/[id] added: admin-gated via
     requireCommunityAdmin, removes the row AND its matching
     publication-state mirror (knyt_publication_states or
     qripto_publication_states based on row.cartridge).
   - KnytCommunityContentAdminTab gains a Delete button (red) next to
     Reject (amber). Confirms before firing — misclick doesn't nuke.
     The previous Reject was the only destructive action; "delete
     button not working" was reported because no delete existed at all.
   - KnytCommunityContentAdminTab gains an optional cartridge prop;
     queue fetch scopes via /api/community-content/list?cartridge=...
   - QriptoPulseAdminTab (new, thin wrapper) renders the admin tab
     with cartridge='qripto'. Admin › Pulse mounts this real
     component instead of the PlaceholderTab.

4) Admin sub-tab labels — drop the word "Admin" (redundant since
   they're all inside the Admin group):
     - "Magazine and Codex Admin"  → "Magazine and Codex"
     - "Pulse Admin"                → "Pulse"
     - "Premium Admin"              → "Premium"
     - "Partners Admin"             → "Partners"
     - "Polity Admin"               → "Polity"
     - "Edit" unchanged

Files:
  app/api/community-content/list/route.ts
    - cartridge query param + filter
    - cartridge column in select + response
  app/api/community-content/[id]/route.ts
    - new DELETE handler (admin-gated, cartridge-aware mirror cleanup)
  app/triad/components/codex/tabs/KnytCommunityContentTab.tsx
    - optional cartridge prop, threaded into list fetch + load deps
  app/triad/components/codex/tabs/KnytCommunityContentAdminTab.tsx
    - optional cartridge prop, threaded into queue fetch + load deps
    - Delete button + handler (with window.confirm)
  app/triad/components/codex/tabs/QriptoPulseTab.tsx (new)
    - thin wrapper, cartridge='qripto'
  app/triad/components/codex/tabs/QriptoPulseAdminTab.tsx (new)
    - thin wrapper, cartridge='qripto'
  app/triad/components/codex/TabRenderer.tsx
    - register QriptoPulseTab + QriptoPulseAdminTab
  data/codex-configs.ts
    - Qriptopia gets 5th tab (admin, adminOnly)
    - Qriptopian Pulse mounts QriptoPulseTab (was PlaceholderTab)
    - Pulse Admin mounts QriptoPulseAdminTab (was PlaceholderTab)
    - Admin tab labels lose the "Admin" suffix

Note: KnytCommunityContentTab and KnytCommunityContentAdminTab still
```
