```
sprint 7 — ens subnames for personas + lockers via namestone

shipped (per 2026-06-13 hackathon plan §sprint 7):

- supabase/migrations/20260613500000_persona_ens_names.sql
  - persona_ens_names: opt-in ENS label per persona. ens_full UNIQUE
    across the platform. namestone_response stored for debug. status
    pending|live|released.
  - locker_ens_names: opt-in ENS label per locker (one active per holder).
  - RLS: holder reads own assignment; service role writes; reverse
    lookup is intentionally bypassed via the public resolver.

- services/identity/namestoneClient.ts — mintEnsSubname +
  resolveEnsSubname. stub mode (no NAMESTONE_API_KEY) emits a
  Namestone-shape response with stub:true so the wallet UI completes.
  live mode hits Namestone public_v1 set-name endpoint.

- POST /api/identity/persona/[id]/ens — spine-auth. ownership check:
  caller must equal the persona_id in the URL. label validated
  ^[a-z][a-z0-9-]{2,40}$. resolves to a deterministic public commitment
  hash (NOT persona_id). returns 409 on duplicate, 503 if migration
  pending.
- GET /api/identity/persona/[id]/ens — returns the live assignment if any.

- GET /api/identity/resolve-ens/[name] — PUBLIC + CORS-enabled reverse
  resolver. accepts bare label OR full name. returns kind ('persona' or
  'locker'), ens metadata, and persona_public_ref (T1-safe commitment).
  NEVER returns persona_id. locker reverse resolution intentionally
  withholds publicRef — only the bound delegate can map via the
  QubeTalk channel.

- scripts/create-env-production.js — adds NAMESTONE_API_KEY,
  NAMESTONE_API_BASE, ENS_PARENT_NAME to allowlist.

anonymity guardrail: even when ENS is claimed, the citizen<->agent
relationship remains obfuscated via DVN T0->T2 + ProveKit. ENS is a
surface label only — it does not collapse the privacy model. operator
note from 2026-06-13.

operator step: run migration 20260613500000_persona_ens_names.sql in
supabase sql editor. set NAMESTONE_API_KEY (from namestone.com) and
optionally ENS_PARENT_NAME (defaults to polity.eth) to leave stub
mode and mint real L2 subnames.
```
