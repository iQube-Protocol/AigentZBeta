## Commit Message

```
fix purchase modal showing sign-in screen to authenticated users

ContentPurchaseModal now uses useSupabaseSessionPersonas to detect the live
Supabase session — a user who is authenticated but whose personaId hasn't
resolved yet no longer sees 'Sign In Required'. Instead:
- isSignedIn = true when either personaId prop is set OR a session email exists
- resolvedPersonaId falls back to sessionPersonas[0].id when prop is absent
- New intermediate state shows 'Signed In' + email when session exists but no
  persona is linked yet (prompts to open wallet instead of sign in again)
- Purchase API calls use resolvedPersonaId so session-backed users can purchase
```
