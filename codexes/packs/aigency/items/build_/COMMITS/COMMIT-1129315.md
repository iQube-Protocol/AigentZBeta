## Commit Message

```
enforce no-hallucination grounding on aigentMe narrative generators

Fixes fabricated progress prose (e.g. "active users rising by 15% over
the past quarter") emitted by strategy inference and reused across the
Brief, Move-forward, and Venture Progress surfaces.

Two-layer enforcement via new services/orchestration/groundingContract.ts:
- PROMPT layer: GROUNDING_MANDATE appended to strategyInference,
  specialistRouter, and nbeLlmRerank system prompts. States the input is a
  DECLARED BASELINE (intent/targets), not achievement; bans invented
  metrics/percentages/counts/dates/trends/names; requires "no data yet"
  over plausible filler.
- VALIDATION layer: groundProse / hasUngroundedQuantifier reject any
  generated sentence introducing a number, percentage, or temporal-trend
  phrase not present in the grounding data. strategyInference now scrubs
  ungrounded prose (headline, venture/personal paragraphs, coherence note,
  correlation explanations) back to the deterministic grounded baseline and
  downgrades confidence to low when it does.
- strategyInference temperature lowered 0.4 -> 0.2.

ventureProgressBuilder is already deterministic (counts only) and needs no
change; this closes the LLM prose vectors that fed it.
```
