## Commit Message

```
intent chains commit 3: dispatcher + advancer + refs + listener hook

The orchestrator core. Implements the full state machine for chain
dispatch and advancement; hooks into the existing emitOrchestrationEvent
so any outcome event flowing through the DVN plane synchronously
advances the matching chain (single in-process listener, no separate
worker).

services/intentChains/refs.ts:
- resolveRef: parses $nbe.X / $prev.X / $chain.X against a chain context
  with __nbe + __prev slots populated by the dispatcher + advancer
- resolveRefsInObject: recursive walk to substitute refs in nested
  body specs (RPC body_static templates etc.)
- evaluateBranch: tiny expression grammar for branch predicates
  (outcome.X truthy, $chain.X == true|false, decision == 'literal').
  Anything unparseable returns false so branches fall through to
  step.next safely

services/intentChains/dispatcher.ts:
- dispatchChain(persona, input): full dispatch flow
  1) template lookup; 2) cartridge_scope guard; 3) triggered_by_nbe
  authorization; 4) Q¢ cost check (v1 stub — wallet integration TODO;
  cost_qc recorded + charge_status='committed' so receipt shape is
  correct); 5) initial context seeding (__nbe slot from input.nbe_seed);
  6) chain row insert with template_version snapshot per spec §11 #5;
  7) intent_chain_started receipt emission (T0 stripped via
  buildChainReceiptMetadata + sanitizer); 8) charge_committed receipt
  when cost_qc > 0; 9) per-kind first-step dispatch returning a
  client UI hint (open_composer / show_pill / rpc_in_flight /
  scheduled / wait)
- DispatchError class with explicit reason codes

services/intentChains/advancer.ts:
- advanceChainIfNeeded(event): the listener hook. Filters out internal
  chain_* events (so internal emissions don't recursively trigger
  themselves), matches the event to the chain's current step via
  expected_outcome_event_type, then advances. Best-effort, never
  throws (errors logged).
- advanceRpcStep: server-to-server fetch with X-Chain-Orchestrator-Token
  + per-step timeout (default 30 min per spec §11 #3); emits
  step_dispatched receipt before the call, marks step_failed on
  non-2xx / timeout / network error
- materializeUserPending: emits intent_chain_step_user_pending so the
  workspace + pill surface can render
- completeUserStep: synthetic outcome path for compose/approve steps
  (advanced via API call from the AigentMeWelcomeSplitTab seam in
  commit 7)
- onStepOutcomeObserved (internal): merges outcome into context
  (with __prev slot + top-level artifact_id / proposal_artifact_id /
  message_id surfacing for $chain.X refs; recipient never written —
  it's T0 PII flowing only through the body of the rpc step that
  uses it); evaluates branches in order; emits step_completed
  (with step.receipt_metadata_keys allowlist enforced); emits
  step_rerouted when a branch takes a non-default path;
  transitions to next step or completeChain on terminal
- transitionToStep: persists new current_step_*, schedules cron
  pickup for scheduled steps, materializes pills for user-facing
  steps, fires advanceRpcStep for rpc, emits step_dispatched receipt
  for scheduled (so audit shows the wait was set up)
- completeChain: marks status='completed', emits intent_chain_completed
  with total_steps + duration_ms
- markStepFailed: emits step_failed with error_class (timeout / http /
  network / config / unknown); halts chain when template.on_failure='halt'
  (default) by emitting intent_chain_failed and status='failed'
- actorToRole helper maps chain actors → AgentRoleId for the event
  from_role/to_role fields

services/orchestration/orchestrationEvents.ts:
- Single addition at end of emitOrchestrationEvent body: if
  event.metadata.chain_id is a string, dynamically import
  advanceChainIfNeeded and call it. Wrapped in try/catch — advancement
  failures don't break receipt durability. Dynamic import avoids
  circular dependency at module load.

End-to-end shape now complete for declarative chain execution.
Smoke-tested refs + branch evaluator inline (vitest sandbox-blocked;
CI runs full suite). Next: commits 4–7 wire the API routes, Marketa
intake, cron extension, and the AigentMeWelcomeSplitTab UI seam.
```
