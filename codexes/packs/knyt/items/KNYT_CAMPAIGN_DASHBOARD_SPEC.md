# KNYT Campaign Dashboard Spec

## Purpose
Define the operating dashboard for the metaKnyt Kickstarter activation campaign.

The dashboard should help Marketa answer four questions quickly:
1. Are we reactivating the right investors?
2. Are we converting attention into Kickstarter backing?
3. Are we turning backers into signal, advocacy, and deeper ecosystem participation?
4. Is this campaign proving the broader AgentiQ / metaMe / Venture Studio model?

---

## Dashboard design principles
- operational first
- show both funnel and loop
- segment-aware
- milestone-aware
- human-readable

---

## Suggested sections
### A — Executive snapshot
Total investors, reactivated investors, Kickstarter visits, Kickstarter backers, investor-tier backers, shares/referrals, Runtime/KNYT follow-through, daily momentum pulse.

### B — Investor reactivation
Dormant / warming / reactivated / engaged / advocates / recruiters by cohort and over time.

### C — Kickstarter conversion
Sent → opened → clicked → visited → backed funnel, by cohort and by channel.

### D — Offer performance
Top KNYT Shelf interest and conversions, Zero KNYT interest and conversions, remaining 21-slot counts, offer confusion or drop-off.

### E — Channel performance
Email, SMS, social, Qriptopian, Runtime/KNYT, Tasks & Rewards.

### F — Participation and signal
Backers re-entering Runtime, KNYT follow-up participation, shares, referrals, community actions, signal actions.

### G — PCS / OM progression
Campaign-state movement, metaMe ladder stages, OM relevance, future creator/correspondent interest.

### H — Partner and Venture Studio proof
Partner outreach count, partner amplifications, joint content, reusable assets, tools/skills ingested, validated loops.

---

## Core filters
- date range
- investor cohort
- campaign state
- message angle
- offer fit
- channel
- investor vs non-investor
- partner-origin traffic
- backed vs not backed

---

## Daily dashboard questions
- Which cohort moved most today?
- Which cohort stalled today?
- Which channel produced the strongest action?
- Which message angle is converting best?
- Are Top KNYT Shelf or Zero KNYT moving faster?
- Are people only clicking, or actually backing?
- Are backers entering Runtime / KNYT follow-through?
- Are we generating advocates and recruiters yet?
- Which investors deserve faster follow-up now?
- Which partners deserve deeper follow-up now?
- Which contacts should be deprioritized for the moment?
- What should change tomorrow?

## Dynamic sequencing rule
The dashboard should not only report results. It should actively support sequencing decisions.

### Canonical rule
**Initial outreach should be sequenced. Follow-up sequencing should become dynamic based on responsiveness, strategic value, and live campaign signal.**

---

## Minimum viable dashboard
### Required immediately
- total investor contacts
- total email sends / opens / clicks
- total SMS sends / clicks
- Kickstarter visits
- Kickstarter backers
- Top KNYT Shelf conversions
- Zero KNYT conversions
- remaining slot counts for both tiers
- reactivated investor count
- share / referral count
- Runtime / KNYT follow-up count if available

### Add next
- cohort-filtered views
- OM / PCS progression views
- partner attribution
- task completion metrics

---

## Alert conditions
Alert Marketa if:
- investor email opens fall below threshold
- clicks are strong but backing is weak
- one investor tier is misunderstood or ignored
- scarcity messaging is not producing action
- Runtime / KNYT follow-through is near zero
- the high-confidence cohort is under-converting
- partner outreach is producing no amplification

---

## Canonical line
**The dashboard should not just show campaign performance. It should show whether the campaign is reactivating investors, converting them now, turning backers into signal and advocacy, and proving the broader ecosystem loop in real market conditions.**
