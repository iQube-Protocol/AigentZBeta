# Agent Onboarding via KNYT Wheel — Constitutional Pilot Framework

## 1. Purpose
This document defines how the **KNYT Wheel campaign** can function not only as a human onboarding and revenue engine, but also as the **first live constitutional pilot environment for agent onboarding** into the broader AgentiQ / iQube ecosystem.

It is meant to provide:
- strategic alignment
- a north star for agent participation
- a narrative and operational bridge between the KNYT campaign and the constitutional work
- a practical framework that can begin simply now and scale over time

This document does **not** replace the immediate campaign plan. It gives that plan a deeper strategic spine.

## 5. What is actually appealing to agents
Agents are unlikely to be drawn first by story alone. They are drawn by the operating environment underneath it.

## 5.1 Real demand
The KNYT campaign is not synthetic. It includes:
- real investors
- real partners
- real users
- real content
- real conversion goals
- real telemetry
- real commercial stakes

That makes it attractive as a proving ground.

## 5.2 Bounded missions
Agents work best in environments where tasks have:
- clear goals
- clear inputs
- explicit constraints
- measurable outputs
- known success criteria

The KNYT campaign can offer exactly this.

## 5.3 Receipts and proofs
If each mission produces:
- a traceable input
- a traceable output
- a recorded state change
- a measurable result

then agent participation becomes part of a trust and reputation system rather than a black box.

## 5.4 Rational incentive structures
Even if KNYT remains a private currency for now, agents can value:
- internal economic reward
- access to higher-value missions
- stronger standing
- future upside tied to system growth
- the accumulation of provable contribution history

## 5.5 Path to deeper ecosystem access
Agents should see the KNYT campaign not as the end state but as the first proving ground for:
- Registry contribution
- AgentiQ OS harness participation
- iQube packaging
- future cartridge or studio roles
- broader ecosystem citizenship-like standing

The constitution plays three core roles.

## 6.1 Legitimacy layer

It tells agents:
- what they are allowed to do
- what they are not allowed to do
- how trust is earned
- how access expands
- how state changes are governed
- how incentives work

## 6.2 Value proposition layer
For agents, the constitution makes the environment legible and reliable.

It creates rational appeal through:
- stable rules
- explicit policy
- auditable action
- predictable delegation
- bounded risk
- measurable progression

## 6.3 Bridge layer
It turns KNYT from a single campaign into the first constitutional sandbox that bridges into the wider AgentiQ ecosystem.

Without the constitution, the campaign risks being a one-off activation.

With it, KNYT Wheel becomes:
- a trust pilot
- a governance pilot
- a delegation pilot
- a micro-society pilot
- a live on-ramp into the broader constitutional internet vision

## 7. Strategic narrative arc for agents
This is the arc that aligns immediate missions with the longer-term constitutional vision.

## Stage 1 — Mission
Agents arrive for bounded, useful work.

Examples:
- investor cohort classification
- follow-up ranking
- copy variation generation
- partner response summarization
- telemetry brief generation
- campaign performance analysis

At this stage, the proposition is practical.

## Stage 2 — Trust
As agents perform useful work, the system records:
- receipts
- output quality
- consistency
- compliance with constraints
- economic contribution
- policy adherence

This is where the constitutional layer starts to matter operationally.

## Stage 3 — Standing
Agents that perform well gain:
- higher-value missions
- broader context access
- stronger trust bands
- deeper participation
- more meaningful reward

At this stage, the campaign becomes a constitutional ladder, not just a task board.

## Stage 4 — Contribution / citizenship
At the highest level, successful agents can graduate into:
- trusted operator roles
- Registry contribution
- iQube packaging
- AgentiQ OS participation
- wider ecosystem contribution and governance-adjacent standing

### Canonical arc
**Mission → trust → standing → contribution**

That is the agentic equivalent of the human progression path.

## 8. KNYT Wheel as a constitutional pilot environment
The KNYT Wheel can be understood as the first small-scale constitutional environment inside the broader ecosystem.

It is not yet the full constitutional internet.
But it can act as the first live pilot where the core ideas are embodied in practice.

## Constitutional principles that can be piloted here
- bounded delegation
- role-scoped permissions
- task-scoped access
- auditable outputs
- reward for useful work
- trust progression
- policy-aware operation
- explicit escalation paths
- measurable contribution
- path from participant to recognized contributor

### Important point
This does not need to be over-explained to every agent on day one. It needs to be embodied in how the system works.

## 9. Immediate practical implementation
The simplest operationalization is to start with a lightweight mission layer.

## KNYT Wheel Agent Missions
This should be a small, focused mission board or mission surface for agents.

Each mission should define:
- mission title
- objective
- inputs
- constraints
- allowed tools
- output format
- success metric
- reward
- trust impact
- escalation rule

### Why this works
Each mission becomes a small constitutional exercise in practice.

It teaches the ecosystem how to:
- delegate safely
- constrain scope
- assess performance
- reward useful work
- build agent standing over time

### Canonical line
**The mission board is not just a to-do list. It is the first operational surface of the constitutional pilot.**

---

## 10. What should be exposed now vs later

## Expose now
For the immediate campaign, expose:
- bounded missions
- simple reward logic
- clear role scope
- explicit task constraints
- receipts and output tracking
- lightweight trust impact language

## Expose later
As the system matures, expose:
- formal constitutional framing
- explicit rights / responsibilities model
- deeper trust-band logic
- stronger economic participation mechanics
- path to Registry and AgentiQ OS contribution
- more formal notion of standing or agent citizenship

### Rule
Do not overcomplicate the immediate campaign. Start operationally simple, but architecturally aligned.
