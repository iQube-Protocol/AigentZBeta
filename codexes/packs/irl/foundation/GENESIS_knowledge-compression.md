## Note on method — Nonnegative Matrix Factorization (nimfa) as a candidate compressor

The operator asked how **nimfa** (`nimfa.biolab.si` — a Python library implementing many Nonnegative Matrix Factorization algorithms; quality measures include RSS, explained variance, KL divergence, and sparseness; applied to gene-expression analysis, network inference, and heterogeneous **data fusion**) might aid knowledge comprehension.

**NMF is a computational realisation of Knowledge Compression.** NMF factors a nonnegative matrix `V ≈ W · H` into two lower-rank nonnegative matrices: `W` a small set of **parts / latent factors**, `H` how each observation is composed from those parts. Because the factors are constrained nonnegative, the decomposition is **additive and parts-based** (unlike PCA's mixed-sign, hard-to-interpret components) — the factors read as *interpretable, recurring building blocks*, i.e. candidate **invariants**. This maps onto the genesis doctrine directly:

- **Knowledge Compression (Stage 3):** run NMF over a domain corpus (or an operator's Experience Matrix) to extract the small `W` of recurring latent factors — an empirical "smallest coherent set" candidate; the chosen **rank** is the compression budget ("3–10, not hundreds").
- **metaQube vs blakQube (two axes):** `H` (how observations map to factors) is a **discoverability** compression; `W` (the factors themselves) is an **understanding** compression — the same two-axis split the dialogue names.
- **Propagation-fidelity / stress test:** NMF's **reconstruction error** (RSS / explained variance / KL) and **sparseness** are quantitative proxies for "does this small set of invariants faithfully regenerate the domain?" — the residual localises exactly where the invariant set is *incomplete* (the personhood→identity drift, made measurable).
- **Data fusion:** nimfa's heterogeneous-dataset fusion mirrors composing multiple KnowledgeQubes into one constitutional context.

**Honest limits:** NMF finds *statistical* latent factors, not *ratified constitutional* invariants — it is a discovery/curation aid for the Curate stage (proposing candidate invariants + measuring their reconstruction fidelity), not a replacement for human ratification (Law XI) or the projection-operator semantics the crystal already defines. It is a tool for the **Knowledge Resolution Engine** (CFS-040) reuse/compose decision and for empirically scoring how much of a domain a candidate invariant set explains — a bridge between the intuition ("curated invariants propagate faithfully") and a number.
