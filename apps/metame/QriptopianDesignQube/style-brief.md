# Qriptopian DesignQube — Style Brief

## Purpose
This is a guidance-level DesignQube derived from the legacy metaKNYT design exports (XD archive + screenshots). It constrains the look/feel and interaction grammar for metaMe Runtime (mobile-first) and Studio (desktop-first) without requiring strict pixel parity.

## Brand primitives
- Light metaMe branding is defined by beige/cream surfaces with coral actions.
- Dark mode uses near-black cinematic surfaces with glass/translucent overlays.
- UI is card-forward and modular: cards + tabs + drawers/sheets carry most secondary complexity.

## Light theme (metaMe beige)
- Use beige/cream as the base background and surface material.
- Coral is the primary action and accent color (CTAs, active underlines, key chips).
- Typography can be modernized, but the light theme should remain warm and minimal.

## Dark theme (KNYT glass)
- Use near-black backgrounds and charcoal surfaces.
- Overlay containers (top bars, bottom nav, drawers, sheets, tab containers) should use glass:
  - blur + alpha + subtle border.
- Maintain a single strong accent (coral) for actions and active state.

## Interaction grammar
- Max 3 primary actions visible at a time.
- Tabs are understated: text tabs with a thin coral underline.
- Use drawers/sheets to contain secondary complexity rather than adding more top-level actions.
- Bottom navigation may be an overlay/center-mode affordance; it should never obscure critical content.

## Value & currency display policy
- Canonical calculations are in Q¢/QCT by default.
- Wallet views prioritize Q¢/QCT.
- $KNYT is reserved for KNYT Offer contexts inside KNYT experiences.
- USD may be shown as a secondary comprehension aid.

## Guidance vs canonical
- This DesignQube is directional. It encodes material rules, color/spacing tendencies, and compositional constraints.
- Where conflicts occur, prefer clarity and trust signals over strict adherence.
